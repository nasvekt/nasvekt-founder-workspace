# NasVekt MVP Roadmap

NasVekt should evolve in three levels.

## Level 1: Prompt Operating System

Manual workflow:

1. Open the dashboard/document.
2. Choose a market area, geography, category, and date window.
3. Run the relevant prompt in the recommended model.
4. Paste the handoff block into the next phase.
5. Continue until the opportunity is killed, modified, validated, or ready to build.

This is the current GitHub MVP. It proves the methodology.

## Level 2: Semi-Automated CLI

Buildable in a week:

- Python CLI asks for founder profile and market filters.
- Runs phases sequentially through model APIs.
- Carries handoff blocks automatically.
- Saves raw output, structured summaries, scores, and gate decisions.
- Generates a final opportunity report.

This proves the system can become software.

## Level 3: Web Dashboard

Full product:

- Market explorer with filters for geography, category, date range, customer type, and goal.
- Pipeline engine for P1 to P4, F1 to F4, C1 to C3, and D1 to D3.
- Evidence vault for sources, claims, signal tiers, and confidence.
- Gate engine for scores, kill thresholds, and next actions.
- Financial dashboard for CAC, LTV, payback, churn, and revenue paths.
- Competitive radar with weekly alerts.
- Deal scanner for agreement analysis and negotiation briefs.

The dashboard should not just display data. It should reduce entrepreneurial uncertainty.
